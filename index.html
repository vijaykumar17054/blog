Understanding the 7 Layers of Data Communication Network

Explore the OSI Model in detail to understand how data communication happens in a network.

Presented by: Vijay

Roll No: 18079

1. Physical Layer

The Physical Layer defines the hardware elements involved in the data transfer. It transmits raw bits over the physical medium like cables, radio, or optical fibers. The primary focus of this layer is on the transmission and reception of the unstructured raw bit stream.

"The physical layer deals with the transmission of data over physical mediums." - OSI Model Reference

2. Data Link Layer

The Data Link Layer ensures reliable data transfer between two devices, providing error detection and correction, and organizing bits into frames for transmission. This layer ensures that the data sent by the physical layer is error-free, and handles retransmission in case of lost data.

Functions: Framing, error detection, and flow control.

"The data link layer is responsible for the correct transfer of data between devices on the same network." - Networking 101

3. Network Layer

The Network Layer handles routing and forwarding of data between different devices across the network. It is responsible for addressing and path selection. In a complex network, this layer enables data to move from one device to another device across multiple networks.

Key Protocols: IP (Internet Protocol), ICMP (Internet Control Message Protocol).

4. Transport Layer

The Transport Layer ensures end-to-end communication between devices. It is responsible for data segmentation, error recovery, and flow control. This layer breaks data into smaller chunks and ensures that they are delivered to the correct application on the receiving device.

Key Protocols: TCP (Transmission Control Protocol), UDP (User Datagram Protocol).

5. Session Layer

The Session Layer establishes, maintains, and terminates communication sessions. It is responsible for managing dialog control between applications. It allows applications on different devices to establish a connection, synchronize data exchange, and control data flow.

Examples: Remote Procedure Calls (RPC), SQL (Structured Query Language) sessions.

6. Presentation Layer

The Presentation Layer formats data for the application layer, ensuring that the data is in a readable form. It handles data encoding, compression, and encryption. This layer is important for ensuring that the data sent by the sender is in a format that the receiver can understand.

"This layer ensures that data is presented in a readable format for the application layer." - OSI Model Explained

7. Application Layer

The Application Layer provides network services to the end-users. It interacts directly with software applications such as email, file transfer, and web browsing. This layer defines the protocols and interfaces for communication between software applications over a network.

Examples: HTTP (HyperText Transfer Protocol), FTP (File Transfer Protocol), SMTP (Simple Mail Transfer Protocol).

"The application layer is the closest layer to the end-user, where software applications interface with the network." - Networking Fundamentals
